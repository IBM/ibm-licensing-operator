## API Compatibility Fixes

The `prometheus-operator/prometheus-operator` package at v0.57.0 introduced three
breaking type changes compared to the old `coreos/prometheus-operator` v0.41.0.
All three required fixes in `controllers/resources/service/service_monitor.go`:

### 1. `Interval` field — new named type
`Duration` is now a distinct named type instead of a plain `string`.

```go
// Before
Interval: interval,  // interval is string

// After
Interval: monitoringv1.Duration(interval),
```

### 2. `SourceLabels` field — new named slice element type
`RelabelConfig.SourceLabels` changed from `[]string` to `[]monitoringv1.LabelName`.

```go
// Before
var sourceLabels []string
sourceLabels = append(sourceLabels, "__name__")

// After
sourceLabels := []monitoringv1.LabelName{"__name__"}
```

### 3. `TLSConfig.CA` — field moved into embedded `SafeTLSConfig`
`TLSConfig` now embeds `SafeTLSConfig` (which holds `CA`). Struct literals must use
the embedded type name explicitly.

```go
// Before
&monitoringv1.TLSConfig{
    CA: monitoringv1.SecretOrConfigMap{...},
}

// After
&monitoringv1.TLSConfig{
    SafeTLSConfig: monitoringv1.SafeTLSConfig{
        CA: monitoringv1.SecretOrConfigMap{...},
    },
}
```
